0- Trabajar en el código:
1- Para subir cambios, ir al bash desde la carpeta de mi proyecto.
2- git status # verificar todo ok(se muestra que se cambió mi archivo). Si hay un archivo .gitignore, ignorará los archivos con las extensiones indicadas en él.
3- git add .
4- git commit -m "mensaje"  # los incisos 3 y 4 añaden todo al repositorio local
6- git push -u origin main # envía los commits al repositorio remoto de nombre "origin" en la rama "main".
	-u <repoRemoto> <rama> permite guardar los parámetros: repoRemoto y rama para luego hacer directamente, (parado en el proyecto)
	git push y que haga el push a repoRemoto y a tal rama.
	sino la otra es indicar siempre el repo y la rama así: git push <	repoRemoto> <rama>
7- git log #muestra los commits realizados

### COMANDOS UTILES ###
git branch # permite ver la ramas disponibles y la rama actual. Si no hiciste nunca un commit, la consola no mostrará ninguna rama.
git branch nombre # crea la rama nombre
git checkout -b nombre # crea la nueva rama "nombre" y te posiciona en ella para trabajar
git checkout nombre # me muevo a la rama nombre
git show-branch # muestra todas las ramas del proyecto con sus commits realizados al momento

git push -u origin nombre # añade los commits realizados a la rama nombre (en el repositorio git)
git remote # me indica el nombre del repositorio en el que estoy parado
